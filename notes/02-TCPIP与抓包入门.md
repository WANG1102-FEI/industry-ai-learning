## 一、TCP/IP 四层模型

自上而下封装数据，自下而上解析数据：

- **应用层**：负责"说什么"（HTTP、MQTT、Modbus TCP 等协议在此层）
- **传输层**：负责"可靠地送给谁"（TCP/UDP，端口号在此层）
- **网络层**：负责"找到路"（IP 地址，路由在此层）
- **网络接口层**：负责"物理发送"（以太网、Wi-Fi 等硬件层）

## 二、TCP 三次握手

三次握手是 TCP 面向连接特性的核心体现，其不是简单的心灵感应确认，而是结合了 **"通信能力测试"**、**"序列号同步"** 与 **"过滤陈旧请求"** 三重目的的技术设计。多出的第三次交互，主要保障了连接建立的唯一性和可靠性。

## 三、Wireshark 抓包实战

- 捕获接口：Adapter for loopback traffic capture
- 捕获目标：本地回环 + 浏览器访问
- 捕获结果：
  - 成功捕获到 HTTP GET 请求
  - 服务器返回 HTTP/1.1 200 OK
  - 请求中包含 token 参数
- 关键观察：在 Wireshark 中可以看到请求的完整路径、Host 头等信息
